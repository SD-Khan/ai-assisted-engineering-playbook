# Example Sprints

This document shows how the framework runs in practice.

---

## Example Sprint Goal
Deliver a small feature end-to-end with high quality and low risk.

---

## Sprint workflow

### 1) Clarify outcome
- define goal
- define success metrics
- define constraints

### 2) Break into tasks
- create 5–15 tasks (small, verifiable)
- define acceptance criteria and tests per task
- sequence dependencies

### 3) Execute tasks
- use structured prompts
- implement small increments
- commit per task

### 4) Review and verify continuously
- run tests per task
- do lightweight architecture checks
- keep changes reviewable

### 5) Close loop with logs
- capture what worked
- capture what failed
- improve prompts/templates

---

## What “done” looks like
- acceptance criteria met
- tests green
- docs updated
- no architecture drift
- verification steps documented
